# Jobs-to-be-Done (JTBD) & Persona Analysis

This artifact documents the core user profiles, their high-level goals ("jobs"), and the context in which they evaluate the portfolio of Sebastian Ortega Auriol.

## 1. User Personas

### Persona A: The High-Volume IT Recruiter (e.g., "Valeria")
- **Role**: Talent Acquisition Specialist / IT Recruiter.
- **Goal**: Quickly find, scan, and filter candidates matching specific stack criteria (React, Node, SQL) and download their CV.
- **Context**: Under high time pressure. Valeria has 50 resumes to review in a morning. She opens portfolios on dual screens (or occasionally on her phone while commuting).
- **Tech Savviness**: Medium. Familiar with tech terms but does not read source code.
- **Success Criteria**: Finds clear contact details and downloads the candidate's CV in under 15 seconds.

### Persona B: The Engineering Lead / Tech Lead (e.g., "Mauricio")
- **Role**: Tech Lead / Senior Systems Architect.
- **Goal**: Evaluate the actual coding practices, system design capabilities, and structural thinking of the developer.
- **Context**: Focused deep work. Mauricio wants to see how the candidate handles databases, code split, and layout performance.
- **Tech Savviness**: High. Evaluates the GitHub repositories and inspects DevTools network/console tabs.
- **Success Criteria**: Inspects clean code structures, solid schema designs, and logs-free browser consoles.

## 3. Incumbent Solutions & Pain Points

| Competitor / Solution | User Friction | Consequence |
| --------------------- | ------------- | ----------- |
| **PDF Resumes Only** | Vague descriptions, no visual proof of skills, hard to verify code quality. | Higher risk of screening unqualified candidates. |
| **Generic Portfolios** | Forced artificial loading screens, bloated animations, broken links, no clear CV download. | Recruiter gets frustrated and closes the tab. |
| **Github Profile Only** | Hard to parse for non-technical recruiters; lacks narrative about soft skills or leadership context. | Recommending manager misses the "discipline/leadership" background. |
